1.0 Diretrizes do Plantão Técnico

Este laboratório reflete um desafio real de infraestrutura de redes. Fomos acionados para auditar um sistema complexo composto por dois switches multicamada de Core/Distribuição e quatro switches de acesso de Base, interconectados por malhas redundantes físicas que operam de forma ineficiente.

Mensagem do Plantão: "Noites assim se repetirão, oportunidades não. Utilize a CLI para auditar as portas designadas, root ports e travar o Load Balanced."

1.2 Condições Encontradas e Depuração dos Logs Inicial e Final

A análise técnica reversa feita a partir das evidências do terminal nos trouxe os seguintes achados operacionais:

A Condição Inicial ("Boi na Linha"):
Nos logs primários de show spanning-tree active, a Root Bridge ativa da rede apontava para o endereço MAC 0005.5E9D.5B17 com prioridade 24577. Esse ID não pertencia a nenhum switch de Core! Isso provou que um switch de acesso da camada de Base roubou a eleição central da topologia devido a um erro crônico de prioridade padrão, deixando a rede lenta e instável.
A Pegadinha das Portas Modulares do Core:
A CLI rejeitava comandos horizontais sequenciais como interface range g0/1 - 4. O comando show interfaces status revelou que esse switch modular expande a numeração mudando o barramento do slot inicial (Gig0/1, Gig1/1, Gig2/1...), forçando o uso estruturado de vírgulas na sintaxe do range.
A Condição Final de Sucesso (Depuração):
Após a aplicação correta dos scripts, os comandos finais de validação registraram vitória absoluta. A VLAN 30 no CORE_1 e a VLAN 40 no CORE_2 exibem a flag de autoridade máxima: This bridge is the root. Todas as interfaces modulares conectadas assumiram o papel de Desg FWD (Designadas), indicando balanceamento perfeito.

1.3 Glossário Técnico Spanning Tree (CCNA)

SW ROOT (Root Bridge):
O switch central eleito como "rei da rede" Spanning Tree. É a partir dele que todos os caminhos mais rápidos são calculados. Ele nunca possui portas bloqueadas.
SW NON ROOT:
Switches subordinados que precisam obedecer às rotas calculadas pelo SW ROOT, aceitando bloqueios preventivos de portas em seus uplinks redundancy para suprimir loops.
Root Port (Porta Raiz / Root FWD):
A porta do switch Non-Root que oferece o caminho mais rápido e de menor custo de velocidade até chegar à Root Bridge.
Alternative Port (Modo BLK / Altn BLK):
Porta reserva identificada em estado de bloqueio lógico. Ela não transmite dados comuns dos usuários, servindo apenas de backup caso o link principal sofra um rompimento.

1.4 Caso de Uso Humanizado: O Equilíbrio das Linhas de Comboio

Para simplificar a engenharia de tráfego, imagine o sistema de transporte ferroviário de uma grande metrópole gerenciado por duas grandes estações centrais: **Estação Alfa (CORE 1)** e **Estação Beta (CORE 2)**, que recebem passageiros vindos de quatro terminais de bairro (BASE 1 a 4).

O Cenário com Boi na Linha (Sobrecarga de Rotas):
Se todos os comboios da cidade (as VLANs) decidissem viajar usando apenas os trilhos que levam à Estação Alfa, a Estação Beta ficaria completamente vazia e ociosa. Pior: se um pequeno terminal de bairro decidisse mandar nas chaves dos trilhos, a malha ferroviária inteira sofreria atrasos bizarros por falta de potência de processamento.
A Engenharia com Load Balancing PVST+:
Nós entramos em ação e ativamos uma regra matemática inteligente de balanceamento: Comboios com numeração Ímpar (VLANs 30, 50, 70...) desembarcam na potente Estação Alfa. Comboios com numeração Par (VLANs 40, 60, 80...) desembarcarão na Estação Beta. Travamos as prioridades dos bairros para que nenhum terminal menor tente roubar o controle. Se uma estação central falhar, a outra assume 100% da malha de comboios sem gerar colisões.

2. Configuração de Troncos e Load Balanced no CORE_1

CORE_1# configure terminal
CORE_1(config)# hostname CORE_1
! Ativação dos Troncos nas portas modulares utilizando vírgulas
CORE_1(config)# interface range gigabitEthernet 0/1 , gigabitEthernet 1/1 , gigabitEthernet 2/1 , gigabitEthernet 3/1
CORE_1(config-if-range)# switchport mode trunk
CORE_1(config-if-range)# exit
! Engenharia Spanning Tree: Raiz das VLANs Ímpares
CORE_1(config)# spanning-tree vlan 30,50,70,90,110,130 root primary
CORE_1(config)# spanning-tree vlan 40,60,80,100,120,140 root secondary
CORE_1(config)# end
CORE_1# write

4. Script de Proteção e Alinhamento nas Bases (BASE_1 até BASE_4)

! Aplicar esta sequência nas quatro bases alterando o Hostname correspondente
BASE_1# configure terminal
BASE_1(config)# hostname BASE_1
BASE_1(config)# interface range g0/1 - 2
BASE_1(config-if-range)# switchport mode trunk
BASE_1(config-if-range)# exit
! Forçar modo Non-Root permanente elevando a prioridade ao teto
BASE_1(config)# spanning-tree vlan 30,40,50,60,70,80,90,100,110,120,130,140 priority 61440
BASE_1(config)# end
BASE_1# write

2.2 Justificativa de Comandos e Arquitetura CCNA

  • interface range gi 0/1 , gi 1/1 ... : Justificado pelo desenho físico modular das placas do Core do simulador, que mudam o barramento sequencial do slot vertical, impedindo a leitura do hífen.
  • spanning-tree vlan [IDs] priority 61440: Comando crucial para rebaixar os switches de acesso. Garante que eles operem para sempre como Non-Root, neutralizando falhas que gerariam o famigerado "boi na linha".
  • root primary (24576) / root secondary (28672): Macros que forçam uma hierarquia determinística e cruzada de backup contínuo por software por instância Per-VLAN.
